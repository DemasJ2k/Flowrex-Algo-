 1.  WHAT IS FLOWREX ALGO?
════════════════════════════════════════════════════════════════════════════════

Flowrex Algo is an autonomous algorithmic trading platform. It combines:

   •  Machine-learning models trained on 7+ years of historical data
   •  Live broker integration (Oanda, cTrader, Tradovate, MT5)
   •  AI-powered supervision via Claude for analysis and autonomous
      risk management
   •  Real-time monitoring through a web dashboard + Telegram notifications

Think of it as a MANAGED EXECUTION LAYER around ML trading signals. You
don't write code — you configure agents, pick symbols, set risk limits,
and the platform handles everything else.




════════════════════════════════════════════════════════════════════════════════
 2.  WHAT FLOWREX DOES
════════════════════════════════════════════════════════════════════════════════

For each trading symbol (XAUUSD, BTCUSD, US30, etc.), Flowrex will:

   1.  Fetch live M5 (5-minute) candlestick data from your broker
       every 60 seconds

   2.  Compute 120 technical features — price patterns, order blocks,
       volume profile, volatility, multi-timeframe alignment, ICT/SMC
       structure, momentum

   3.  Run a 3-model ML ensemble (XGBoost + LightGBM + CatBoost) trained
       specifically for that symbol

   4.  Generate BUY / SELL / HOLD signals with a confidence score

   5.  Apply risk filters — news filter, session filter, drawdown limits,
       cooldown

   6.  Place orders on your broker with configured position sizing

   7.  Monitor exits — TP/SL hits, max-hold time, broker reconciliation

   8.  Analyze every trade via the AI Supervisor and report to Telegram

All fully automated. You can pause, adjust, or stop any agent at any time.

─── 4.3  PROP FIRM COMPATIBILITY ─────────────────────────────────────────────

   Prop firms (FTMO, MyForexFunds, TopStep) typically use cTrader or MT5.

   When creating an agent for a prop account:
     •  Set broker to  ctrader  or  mt5
     •  Enable PROP FIRM MODE in the agent config
     •  This enforces tiered drawdown limits:
            yellow  =  -1.5%  →  reduce position size
            red     =  -2.5%  →  stop new entries
            hard    =  -3.0%  →  close everything


─── 4.4  QUICK RECOMMENDATIONS ───────────────────────────────────────────────

     •  ES (S&P 500)               →  Tradovate (cheapest, native futures)
     •  XAUUSD, forex              →  Oanda (tight spread) or cTrader (prop)
     •  BTCUSD, ETHUSD             →  Oanda (CFD, no exchange needed)
     •  US30 / NAS100              →  Oanda CFDs (flexible) or Tradovate
                                      futures (cheaper long-term)

 6.  HOW THE MODELS PAGE WORKS
════════════════════════════════════════════════════════════════════════════════

─── 6.1  UNDERSTANDING THE MODEL LIST ────────────────────────────────────────

Each row = one trained ML model for a specific (symbol, algorithm) pair.

   ┌──────────────┬──────────────────────────────────────────────────────┐
   │  COLUMN      │  MEANING                                             │
   ├──────────────┼──────────────────────────────────────────────────────┤
   │  Symbol      │  Trading pair (XAUUSD, US30, etc.)                   │
   │  Type        │  Algorithm: xgboost / lightgbm / catboost            │
   │  Grade       │  A (excellent) / B (good) / C (marginal)             │
   │              │  D (poor) / F (unusable)                             │
   │  Sharpe      │  Risk-adjusted return — higher = better              │
   │              │    >2  = good                                        │
   │              │    >4  = great                                       │
   │              │    >10 = exceptional                                 │
   │  WR          │  Win rate % on out-of-sample data                    │
   │  Trades      │  Number of trades in the OOS evaluation              │
   │  Last        │  Timestamp the model was created                     │
   │  Trained     │                                                      │
   └──────────────┴──────────────────────────────────────────────────────┘

ENSEMBLE BEHAVIOR:
A live agent loads all 3 model types (xgboost + lightgbm + catboost)
and requires 2-of-3 majority agreement on direction before generating
a signal. All-3 agreement gets +5% confidence bonus.


─── 6.2  WHEN TO RETRAIN ────────────────────────────────────────────────────

     •  After 4-6 weeks of live trading (market regimes drift)
     •  After a major feature pipeline change
     •  When live WR drops significantly below backtest WR
     •  When a new symbol is added


─── 6.3  HOW TO RETRAIN ─────────────────────────────────────────────────────

   1.  ONE SYMBOL — click the Retrain button next to the symbol.
       Runs in background (~1-2h). Progress visible in retrain history.

   2.  ALL SYMBOLS — click Retrain All. Runs sequentially (~8-12h total).

   3.  SCHEDULED — enable monthly auto-retrain. Runs on 1st of each
       month at 00:00 UTC.


─── 6.4  WHAT HAPPENS DURING RETRAINING ─────────────────────────────────────

   1.  Load historical data (up to 500k bars of M5)
   2.  Compute 120 features per bar
   3.  Create triple-barrier labels (TP hit / SL hit / timeout)
   4.  Walk-forward validation — 4 folds
   5.  Hyperparameter search (Optuna, 15 trials per fold per model)
   6.  Train final models on 100% of training data
   7.  Evaluate on out-of-sample (OOS) data
   8.  Only save if Grade >= B  (quality gate)
   9.  Archive old models before overwrite (safety)
  10.  Hot-reload in live agents (no restart needed)


─── 6.5  THE RETRAIN GATE ───────────────────────────────────────────────────

Models only get deployed if they pass automatically:

     •  Grade must be  A  or  B
     •  OOS Sharpe must be  > 2.0
     •  No fold worse than Grade D  (walk-forward consistency)
     •  OOS trade count  > 50  (statistical significance)

If a retrain fails the gate, the OLD MODEL STAYS in place and the
retrain history shows the failure reason. Your production never
breaks from a bad retrain.

 7.  HOW THE BACKTEST PAGE WORKS
════════════════════════════════════════════════════════════════════════════════

─── 7.1  PURPOSE ────────────────────────────────────────────────────────────

Backtest a strategy against historical data BEFORE deploying an agent
or changing config. The engine uses the same feature pipeline + models
as the live agent, so results reflect what would have happened historically.


─── 7.2  CONFIGURATION ──────────────────────────────────────────────────────

   ┌─────────────────────┬────────────────────────────────────────────────┐
   │  SETTING            │  MEANING                                       │
   ├─────────────────────┼────────────────────────────────────────────────┤
   │  Symbol             │  Which pair to test                            │
   │  Agent type         │  flowrex_v2 (3-model ensemble)                 │
   │                     │  or potential (institutional features)         │
   │  Date range         │  Historical window (default: last 2 years)     │
   │  Sizing mode        │  Risk %  or  Max Lots                          │
   │  Risk per trade     │  % of account risked per trade (default 0.10%) │
   │  Max daily loss     │  Stop trading for the day if exceeded          │
   │  Cooldown bars      │  M5 bars to wait after a trade                 │
   │  News filter        │  Skip trades 30min before high-impact news     │
   │  Session filter     │  Only trade during prime hours for the symbol  │
   └─────────────────────┴────────────────────────────────────────────────┘


─── 7.3  RUNNING A BACKTEST ─────────────────────────────────────────────────

   1.  Pick symbol, agent type, date range
   2.  Adjust risk parameters (or leave defaults)
   3.  Click Run
   4.  Watch progress in real-time
   5.  Results appear in 30-60 seconds for a 2-year window


─── 7.4  READING RESULTS ────────────────────────────────────────────────────

   ┌──────────────────┬────────────────────────────────────────────────┐
   │  METRIC          │  INTERPRETATION                                │
   ├──────────────────┼────────────────────────────────────────────────┤
   │  Grade           │  A/B/C/D/F overall quality                     │
   │  Sharpe          │  Risk-adjusted return  —  >2 is good           │
   │  Win Rate        │  % of winning trades                           │
   │  Total Trades    │  More = more statistical confidence            │
   │  Total Return    │  Absolute % return over the period             │
   │  Max Drawdown    │  Biggest equity dip                            │
   │                  │  Lower = better; <10% acceptable               │
   │  Profit Factor   │  Gross profit / gross loss; >1.5 is good       │
   └──────────────────┴────────────────────────────────────────────────┘


─── 7.5  READING THE EQUITY CURVE ───────────────────────────────────────────

Look for:

     •  Smooth upward slope  =  consistent strategy  (GOOD)
     •  Step pattern         =  a few big wins dominate  (RISKY)
     •  Choppy / sideways    =  strategy isn't adding edge
     •  Big drawdowns        =  risk management needs tightening


─── 7.6  COMMON EXPERIMENTS ─────────────────────────────────────────────────

Try running the same symbol with different configs:

     •  Change risk:  0.1% → 0.5% → 1%  →  see how Sharpe changes
     •  Toggle news filter on/off       →  measure impact
     •  Adjust SL multiplier:
            ATR×0.8  →  ATR×1.2  →  trade-off win rate vs avg loss
